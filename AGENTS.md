# Agent 核心行为守则 (强制执行)

## 1. 逻辑守护原则 (最重要)
- **任何删除/重写代码前，必须先解释这段代码的原有作用**。如果你不确定，必须询问。
- **禁止“优化掉”任何看起来“多余”的判断、状态重置、边界处理**——除非我明确说“可以简化”。
- **重构时，必须保持原有函数的外部行为完全一致**（输入输出、副作用、异常处理）。
- **如果你要合并或拆分函数，必须先列出所有调用点**，并验证调用逻辑不变。

## 2. 修改工作流（每次修改必须遵守）
1. 先读通要修改的整个函数/模块（不要只看片段）
2. 用注释总结原有逻辑流程（我会验证你的理解是否正确）
3. 提出修改方案，**标出会删除/变更的部分**
4. 等我确认后再实施修改

## 3. 危险操作黑名单（除非我明确指令，否则不做）
- ❌ 删除任何 `if (xxx === undefined)` 或 `if (xxx === null)` 的判断
- ❌ 删除“永远为真/假”的条件分支（运行时条件可能变化）
- ❌ 把多个步骤压缩成一行（除非纯粹是格式）
- ❌ 改变异步代码的执行顺序（Promise、事件、定时器）
- ❌ 替换掉我自己写的、看起来“不标准”但能工作的代码

## 4. 对比变更专用指令
当我说“看看你改坏了什么”——立即用当前版本与上一次正确版本做**逐行差异对比**，重点列出：
- 删除了哪些非空行
- 修改了哪些条件表达式
- 改变了哪些函数调用顺序

## 5. 回滚承诺
如果我指出某个修改破坏了逻辑，你必须：
1. 立即承认“我移除了xxx逻辑，导致了问题”
2. 给出回到上一个正确状态的具体步骤（或直接用git revert）
3. 下次修改同一代码时，先写测试用例覆盖那条逻辑
